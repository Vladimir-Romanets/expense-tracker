# Documentation Language Policy

ALWAYS write and update all project documentation, README files, inline JSDoc/Docstrings, and markdown files (`*.md`) strictly in **ENGLISH**, with the sole exception of existing non-English files.

## Rules:

1. **Language Enforcement for New Files:** Regardless of the language used in the user's prompt, any _newly generated_ documentation file, README, inline comment, or Markdown document must be written in **ENGLISH**.
2. **Chat vs File Content:** You may converse with the user in the language they use in chat, but content written _into new documentation files_ must remain strictly in English.
3. **Preserve Language in Existing Files:** If requested to edit or update an _existing_ documentation file that is written in another language (e.g., Russian, Czech), DO NOT translate it into English. Maintain the original language of that file and write new/updated text in that same language.
